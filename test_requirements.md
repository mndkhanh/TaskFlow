# INTERN DEVELOPER ASSESSMENT

## Task Description

Build a **Task Management Application (Todo List)**.

## Requirements

### 1. Functional Requirements

- **View Tasks:** Display a list of all existing tasks.
- **Create Task:** Add new tasks to the list.
- **Edit Task:** Modify the details of an existing task.
- **Delete Task:** Remove tasks from the list.
- **Toggle Status:** Mark tasks as completed or uncompleted.
- **Filter & Search:** Allow users to search for specific tasks or filter them based on their completion status.

### 2. Technical Requirements

- **Tech Stack:** Use any programming language and framework you are most proficient with.
- **Code Quality:** Organize the source code clearly; ensure it is highly readable and easy to maintain.
- **Validation:** Properly handle edge cases and invalid input data.
- **Documentation:** Include a `README.md` file with clear instructions on how to install and run the project locally.

### 3. Bonus Points (Optional / Highly Encouraged)

- **Data Manipulation:** Implement pagination or data sorting features.
- **UI/UX:** Ensure the user interface is fully responsive across different screen sizes.
- **Containerization:** Use Docker to containerize the application.
- **Testing:** Write Unit Tests for core logic.
- **Deployment:** Deploy the application to a live online environment (e.g., Vercel, Netlify, Render, AWS, etc.).

---

## Submission Guidelines

Please submit your assessment containing the following deliverables:

1. **Source Code:** The complete source code of your application.
2. **README File:** Setup and execution instructions.
3. **GitHub Repository Link** (if applicable).

**Duration:** 02 days from the assignment date.

> 💡 **Note on Evaluation:** _We value your problem-solving mindset, code quality, project organization, and how you handle edge cases and unexpected scenarios, rather than just checking off the basic functional requirements._
